# Reproducibility notes

This document summarizes how to reproduce the documented UIT_HighDefinition Caption Prediction workflow when authorized ImageCLEFmedical data and the required model access are available.

## 1. Scope

This repository is a paper companion and reproducibility repository. It documents the submitted pipeline but does not redistribute restricted datasets, dataset images, ground-truth annotations, full generated test captions, final submission files, machine-level rank outputs, or private execution logs.

## 2. Prepare official data

Obtain the ImageCLEFmedical Caption 2026 data through the official task channels. Place the test images in an authorized local or Kaggle input path and update the placeholders described in [`PATH_CONFIG.md`](PATH_CONFIG.md).

## 3. Auxiliary CUI predictions

The auxiliary CUI predictions used in this work were generated by running the public UIT_Worker Concept Detection notebook:

```text
https://github.com/nhanbui246/ImageCLEFmedical-2026-UIT-Worker
```

The Concept Detection training and inference code was not modified in this work. Its output was used only as an external auxiliary source of predicted UMLS CUIs.

Expected auxiliary CUI format:

```csv
ID,CUIs
image_id_1,C0000001 C0000002
image_id_2,C0000003
```

The predicted CUIs are mapped to readable biomedical terms and inserted into the Qwen3-VL prompt as optional hints. They are not treated as ground-truth labels.

## 4. Caption inference

The final Caption Prediction system used `Qwen/Qwen3-VL-8B-Instruct` in inference-only mode with 4-bit NF4 quantization.

Key settings:

```text
MODEL_ID = "Qwen/Qwen3-VL-8B-Instruct"
MAX_NEW_TOKENS = 80
MIN_PIXELS = 256 * 28 * 28
MAX_PIXELS = 512 * 28 * 28
do_sample = False
world_size = 6
save_every = 1
```

The sanitized Kaggle notebooks under `caption_inference/notebooks/` document the execution workflow. Private Kaggle dataset slugs have been replaced with placeholders.

## 5. Rank sharding

All test image IDs are sorted by numeric suffix. Rank assignment uses deterministic modulo sharding:

```python
rank = index % 6
```

Rank allocation:

```text
Machine 1: rank 0 and rank 1
Machine 2: rank 2 and rank 3
Machine 3: rank 4 and rank 5
```

Each rank writes an independent CSV file. The notebooks save progress after every generated caption, which supports resume after runtime interruption.

## 6. Caption cleaning

Raw generated captions are cleaned by conservative post-processing. The cleaner removes boilerplate prefixes, normalizes modality terms, controls length, repairs selected truncated endings, filters generic captions, and enforces final punctuation.

See:

```text
caption_inference/cleaner.py
```

## 7. Merge and internal validation

After six rank files are available, merge and validate the outputs:

```bash
python caption_inference/merge_and_validate_submission.py \
  --rank_dir /path/to/qwen3_full_6gpu \
  --test_image_root /path/to/test/images \
  --output_dir /path/to/qwen3_final_submission
```

The final submission CSV must have exactly two columns:

```csv
ID,Caption
```

Internal validation checks include:

- expected row count;
- duplicate IDs;
- empty captions;
- missing IDs;
- extra IDs;
- final ZIP content.

The final submitted file was internally validated and contained 15,249 rows, 0 duplicate IDs, 0 empty captions, and 13,666 unique captions.

## 8. Official checker

The official ImageCLEFmedical 2026 evaluation/checking repository can be used for independent submission-format validation:

```text
https://github.com/taubsity/clef-caption-evaluation-2026
```

In this repository, we document internal validation rather than claiming that the official checker was run as part of the public release.
