# UIT_HighDefinition ImageCLEFmedical Caption 2026

This repository accompanies the CLEF 2026 working notes paper:

**UIT_HighDefinition at ImageCLEFmedical Caption 2026: Soft CUI-Guided Qwen3-VL Captioning**

It is a paper companion and reproducibility repository for the **UIT_HighDefinition** submission to the **ImageCLEFmedical Caption 2026 Caption Prediction** subtask.

The source code, prompt template, post-processing scripts, and reproducibility documentation are available at:

```text
https://github.com/quytsua/UIT_HighDefinition_ImageCLEFmedical_Caption_2026
```

## Repository scope

This repository provides the captioning prompt, post-processing logic, merge and validation scripts, notebooks, environment notes, and result summaries. It does **not** redistribute ImageCLEFmedical dataset images, ground-truth annotations, full generated test captions, machine-level rank outputs, final submission files, private execution logs, or the full official leaderboard CSV.

## Method overview

The submitted system uses a soft concept-guided captioning pipeline:

1. Auxiliary UMLS Concept Unique Identifier (CUI) predictions are obtained by running the public UIT_Worker Concept Detection notebook.
2. Predicted CUIs are mapped to readable biomedical terms.
3. `Qwen/Qwen3-VL-8B-Instruct` generates captions using the mapped terms as optional clinical hints.
4. A conservative rule-based cleaner normalizes output style.
5. Six rank-level outputs are merged and internally validated before submission packaging.

The predicted CUIs are auxiliary predictions only. They are not treated as ground-truth labels and are not forced into the generated captions. The prompt explicitly instructs the model to use the CUI hints only when visually supported by the image.

## Important note on the auxiliary CUI source

The auxiliary CUI predictions were generated by running the public UIT_Worker Concept Detection notebook:

```text
https://github.com/nhanbui246/ImageCLEFmedical-2026-UIT-Worker
```

In this work, the Concept Detection training and inference code was not modified. Its generated CUI predictions were used only as an external auxiliary input for the Caption Prediction pipeline.

See [`THIRD_PARTY_NOTICES.md`](THIRD_PARTY_NOTICES.md) and [`concept_detection/README.md`](concept_detection/README.md).

## Public release policy

The public repository includes code, notebooks, prompt templates, post-processing scripts, validation scripts, and summary documentation. It intentionally excludes:

- full final `submission.csv` files;
- final submission ZIP files;
- machine-level rank CSV outputs;
- full generated captions by test ID;
- private Kaggle execution logs;
- official datasets or images;
- the full official leaderboard CSV.

This policy keeps the repository useful for review and reproducibility while avoiding redistribution of restricted data or full generated test predictions.

## Caption generation settings

| Setting | Value |
|---|---|
| Captioning model | `Qwen/Qwen3-VL-8B-Instruct` |
| Fine-tuning | None; inference only |
| Quantization | 4-bit NF4 via `bitsandbytes` |
| Compute dtype | `float16` |
| Double quantization | Enabled |
| Decoding | Deterministic greedy decoding |
| Sampling | `do_sample=False` |
| Maximum new tokens | `80` |
| Minimum pixels | `256 * 28 * 28` |
| Maximum pixels | `512 * 28 * 28` |
| Maximum CUI terms in prompt | `8` |
| Maximum cleaned caption length | `55` words |

## Distributed inference

The official test set was processed using six deterministic inference ranks distributed across three Kaggle GPU runtimes.

| Machine | Ranks |
|---|---|
| Machine 1 | 0 and 1 |
| Machine 2 | 2 and 3 |
| Machine 3 | 4 and 5 |

Image IDs were sorted by numeric suffix and assigned to ranks by modulo sharding:

```python
rank = index % 6
```

Each rank wrote an independent CSV and saved progress after each image to support safe resume.
